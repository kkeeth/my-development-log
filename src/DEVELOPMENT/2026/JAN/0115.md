# 0115

## LOGS

- AWS Redshift の `no upfront` とは
  - Redshift Serverless 向けの予約（Serverless Reservations）における支払いオプションの一つ
    - 初期費用を前払いせず（upfront が前払いの意味）
    - 1 年間の RPU（Redshift Propcessing Unit）利用を予約でき
    - オンデマンド料金の約 20%の割引が適用
  - upfront だと予測可能な使用量に対して RPU レベルを指定し、予約分を超える使用量はオンデマンド料金で課金される
  - 約 24%の割引だが，全額前払い（柔軟性は高い）
- `Savings Plans` とは 3 つのタイプがある
  - Compute Savings Plans: EC2、Lambda、Fargate に適用可能で最大 66%割引。インスタンスファミリーやリージョンを問わず柔軟
  - EC2 Instance Savings Plans: EC2 限定で最大 72%割引。特定ファミリーに限定
  - SageMaker Savings Plans: SageMaker 向け
  - 支払いオプション 3 通りから選択可能で，長期・前払いほど割引率が高くなる
    - 全額前払い（最大割引）
    - 一部前払い
    - 前払いなし（No Upfront）
  - `RI（Reserved Instances）` との違い
    | 項目 | Savings Plans | Reserved Instances |
    | ------------ | ------------------------------ | -------------------------- |
    | 柔軟性 | 高（複数サービス・リージョン） | 低（特定インスタンス限定） |
    | 対象サービス | EC2/Lambda/Fargate など | EC2/RDS/Redshift など |
    | 割引率 | 最大 72% | 最大 72%（同等） |
- Aurora PostgreSQL の RI（Reserved Instances）で利用可能な R7g インスタンスクラスは，AWS `Graviton3` プロセッサを基盤とした Arm アーキテクチャを使用している
  - `Graviton3`
    - AWS が自社開発した Arm ベースの CPU
    - 前世代 Graviton2 比で最大 30%の性能向上を実現し
    - DDR5 メモリを搭載してメモリ帯域幅が 50% 向上
    - ネットワーク帯域も最大 30 Gbps（EBS は 20 Gbps）と高速化されている
